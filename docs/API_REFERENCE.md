## Database Schema

The project still uses `waims_demo.db` as the generated demo database.

### `players`

Holds anonymized athlete metadata.

Typical columns:
- `player_id`
- `name`
- `position`
- `age`

### `wellness`

Daily subjective monitoring data.

Typical columns:
- `player_id`
- `date`
- `sleep_hours`
- `sleep_quality`
- `soreness`
- `stress`
- `mood`

### `training_load`

Daily external load data.

Typical columns:
- `player_id`
- `date`
- `practice_minutes`
- `game_minutes`
- `player_load`
- `accel_count`
- `decel_count`

### `force_plate`

Neuromuscular monitoring data.

Typical columns:
- `player_id`
- `date`
- `cmj_height_cm`
- `rsi_modified`
- `asymmetry_percent`

### `injuries`

Availability and injury-log data.

### `acwr`

Acute-chronic workload ratio and supporting load fields.

### `availability`

Availability status by athlete and date.

### `ml_predictions`

Generated by `train_models.py`.

Typical columns:
- `player_id`
- `date`
- `injury_risk_score`
- `readiness_score`

## Main Functional Areas

### `generate_database.py`

Creates the synthetic demo database used by the monitoring workflow.

Responsibilities:
- create schema
- generate anonymized player rows
- generate wellness/load/force-plate demo data
- populate injuries and availability

### `train_models.py`

Builds demo model artifacts and derived outputs.

Responsibilities:
- join the synthetic datasets
- engineer features
- train injury-risk model
- calculate readiness outputs
- export model artifacts and processed tables

### `auth.py`

Controls:
- demo login
- role assignment
- visible tabs
- field-level access rules

### `coach_command_center.py`

Renders the main readiness/briefing experience for the larger app.

### `athlete_profile_tab.py`

Renders athlete-level details such as:
- readiness summary
- radar or profile visuals
- trends and context

## InnerAthlete MVP Modules

### `waims_bio/biomarker_tab.py`

Current responsibilities:
- render biomarker gauges
- show optimal ranges
- map marker values into status states
- display action-plan guidance
- screen uploaded CSVs for likely identifiers

### `waims_bio/cognition_tab.py`

Current responsibilities:
- render S2-style cognition score bands
- show metric-by-metric interpretation
- map scores into coaching translation

### `waims_bio/genomics_tab.py`

Current responsibilities:
- render genomics summary by domain
- connect those domains to practical recommendations

### `waims_bio/privacy.py`

Current responsibilities:
- parse uploaded CSVs
- detect suspicious identifier columns or values
- support privacy guardrail copy

### `waims_bio/mvp_content.py`

Shared anonymized MVP content used by the InnerAthlete tabs.

## Privacy Rules For Integrations

Any future import or API work should assume:
- no real athlete names in the repo
- no DOB
- no report identifiers tied to individuals
- no private vendor tokens in committed files

Use anonymized identifiers such as:
- `ATH-001`
- `2024_lr_001`
